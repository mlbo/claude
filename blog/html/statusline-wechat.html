我和 Claude Code 一起写了个状态栏

最近用 Claude Code 时总有个烦恼：聊着聊着上下文就满了，然后被迫压缩对话，之前的上下文就丢了。想着要是能实时看到上下文使用情况就好了。

Claude Code 的回答

我问 Claude Code：「帮我装一个可以看到上下文消耗的插件」。

它没有直接给我装插件，而是告诉我它有内置的 /statusline 功能。

机制很简单：Claude Code 把会话数据打包成 JSON，传给你配置的脚本。你的脚本处理后输出什么，就显示什么。

开始实现

我先让 Claude Code 写了个基础版本，能显示模型名和百分比。跑起来后，我陆陆续续提需求：

• 加上当前目录
• 显示 git 分支
• 运行时长也加上
• 成本不用了
• 显示 tokens 数量

Claude Code 每次都很配合地改。

踩了一个坑

我想加个进度条，用 █ 和 ░ 这两个 Unicode 字符。

macOS 上测试没问题，但在 Linux 服务器上变成了乱码。

排查后发现是 tr 命令的问题——它是按字节处理字符的，遇到多字节的 UTF-8 字符就会出错。

解决方案是用循环构建字符串，而不是用 tr 转换。

最终效果

[Opus] 📁 myproject  git:(main) | ⏱️ 3h8m4s
███░░░░░░░ 35% | 53k/200k | out: 12k | 42.1 t/s

第一行：模型、目录、分支、时长

第二行：进度条、上下文使用、输出 tokens、生成速度

进度条颜色会变：绿色正常，黄色警告，红色危险。

开源实现现状

社区已经有比较成熟的项目：claude-hud by Jarrod Watts

这是目前最完善的状态栏插件，支持工具追踪、子代理状态、Todo 进度条等。

/plugin marketplace add jarrodwatts/claude-hud
/plugin install claude-hud

我的实现更轻量，就是一个独立的 shell 脚本。

一些思考

整个实现过程大概半小时。我只需要描述需求，Claude Code 负责写代码、测试、调试。

有意思的是，它不是给我推荐一个现成的插件，而是告诉我 Claude Code 本身就有这个能力。

Claude Code 的设计哲学是提供机制而不是提供产品——给你一个管道，你想显示什么自己写。

安装

如果你想用：

git clone https://github.com/mlbo/claude.git ~/claude
cd ~/claude/statusline && ./install.sh

文件会安装到 ~/.claude/statusline/ 目录。
